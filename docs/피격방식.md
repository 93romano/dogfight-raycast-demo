  현재 구현된 시스템은 즉시 히트스캔(Hitscan) 방식입니다:

  WeaponSystem.ts:50-66에서:
  1. 카메라 위치에서 방향으로 raycaster.set() 설정
  2. raycaster.intersectObjects() 즉시 충돌 검출
  3. 발사 버튼을 누르는 순간 바로 히트 판정

  VisualBullet.ts의 시각적 총알은 단순히 시각 효과일 뿐, 실제 히트 판정과는 무관합니다.

  이는 다음과 같은 특징이 있습니다:
  - ✅ 레이턴시 없음 - 조준한 곳에 즉시 맞음
  - ✅ 정확한 조준감 - 마우스 포인터 = 히트 지점
  - ✅ 구현 단순 - 복잡한 물리 계산 불필요
  - ❌ 비현실적 - 실제 총알 궤적과 다름
  - ❌ 회피 불가 - 발사 순간 무조건 명중

  FPS 게임에서는 일반적으로:
  - 히트스캔: 저격총, 권총 등
  - 투사체: 로켓, 수류탄 등

  현재 시스템이 도그파이트 게임에 적합한지 검토가 필요할 수 있습니다.

⏺ 히트스캔 vs 투사체 시스템 검토

  현재 게임 특성 분석

  도그파이트 게임의 특징:
  - 고속 비행 (200+ units/second)
  - 3D 공중전
  - 실시간 멀티플레이어
  - 항공기 간 전투

  1. 히트스캔 시스템 (현재)

  장점:
  - ✅ 즉시 반응 - 네트워크 지연 영향 최소
  - ✅ 예측 가능 - 조준한 곳에 정확히 명중
  - ✅ 구현 단순 - 복잡한 물리 계산 불필요
  - ✅ 성능 우수 - CPU 부하 적음

  단점:
  - ❌ 비현실적 - 실제 기관포/미사일과 다름
  - ❌ 회피 불가 - 스킬 기반 플레이 제한
  - ❌ 거리 제한 - maxShotRange: 1000으로 인위적 제한

  2. 투사체 시스템 (대안)

  장점:
  - ✅ 현실적 - 실제 탄도학 반영
  - ✅ 전술적 깊이 - 선행 조준, 회피 가능
  - ✅ 시각적 만족감 - 탄궤 추적 가능
  - ✅ 다양한 무기 - 속도별 특성화 가능

  단점:
  - ❌ 네트워크 복잡성 - 동기화 어려움
  - ❌ 예측 조준 필요 - 초보자 접근성 저하
  - ❌ 성능 부하 - 물리 계산 증가
  - ❌ 지연 영향 - 레이턴시에 민감

  3. 권장사항

  도그파이트 게임에서는 투사체 시스템이 더 적합합니다:

  1. 게임성: 선행 조준과 회피 기동이 핵심 스킬
  2. 몰입감: 실제 항공기 전투와 유사한 경험
  3. 전술 다양성: 거리별 명중률 차이로 전술적 선택 증가

  구현 제안:
  - 기관포: 빠른 투사체 (500-800 units/s)
  - 미사일: 유도 투사체 (추후 확장)
  - 히트스캔은 레이저 무기 등으로 한정

⏺ 탄약 시스템 설계

  핵심 요구사항

  - 탄약 수: 30발
  - 재장전: R키, 3초 소요
  - 0발 시: 자동 발사 불가, 수동 재장전 필요

  시스템 설계

  WeaponSystem 확장:
  interface WeaponStatus {
    isReady: boolean;
    cooldownRemaining: number;
    shotsFired: number;
    // 추가
    ammo: number;           // 현재 탄약
    maxAmmo: number;        // 최대 탄약 (30)
    isReloading: boolean;   // 재장전 중 여부
    reloadTimeRemaining: number; // 재장전 남은 시간
  }

  상태 전환:
  1. 발사 가능: ammo > 0 && !isReloading && cooldownOk
  2. 재장전 중: isReloading = true, 발사 불가, 3초 타이머
  3. 재장전 완료: ammo = maxAmmo, isReloading = false

  UI 표시:
  - 탄약 카운터: "30/30"
  - 재장전 프로그레스바
  - 빈 탄창 시 경고 표시

  네트워크:
  - 발사 시: 탄약 차감 동기화
  - 재장전 시: 상태 브로드캐스트

  입력 처리:
  - 마우스 클릭: shoot() - 탄약 체크 포함
  - R키: reload() - 재장전 시작
